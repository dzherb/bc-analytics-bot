## Инструкция по запуску

- Создаем файл `.env` по примеру `.env.template`. Понадобится токен телеграм бота, его можно создать через [BotFather](https://t.me/BotFather) 
- Убеждаемся, что в системе установлен пакетный менеджер [uv](https://docs.astral.sh/uv/) и make (по-умолчанию идет с Linux и MacOS)
- Выполняем команду
    ```bash  
    make runserver
    ```

## Структура проекта

- `docker` - Dockerfile для всех точек входа
- `entrypoints` - точки входа для запуска основных частей приложения (пока это только телеграм-бот, потом может появиться воркер для работы с файлами или что-то еще)
- `infra` - инфраструктурный код: управление настройками, конфигурация логгирования, там же может быть подключение к кэшу, базе, брокеру и т.п. 
- `models` - доменные модели приложения, ими оперируют сервисы
- `scripts` - различные вспомогательные скрипты
- `services` - сервисы с бизнес-логикой, центральная часть всего приложения
- `telegram_bot` - логика взаимодействия бота с пользователем, по-сути, весь ui
- `tests` - тесты и фикстуры

## Линтеры

### Запуск всех линтеров и проверок

```bash
make fullcheck
```


Также можно запустить проверки по-отдельности:

### Соответствие style guide

```bash
make lint
```

### Типизация

```bash
make typecheck
```

### Линтинг `.env` файла

```bash
make lint-dotenv
```
